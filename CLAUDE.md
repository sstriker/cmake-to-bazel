# Project conventions for Claude Code

## Commit messages and PR descriptions

Do NOT include the auto-generated session-tracking URL of the form
`https://claude.ai/code/session_<id>` in any commit message, PR description,
PR comment, or PR review. These URLs are private to the operator's Anthropic
account and should not be embedded in shared repo history or public PR
metadata.

A neutral footer like `_Generated by Claude Code_` (without a URL) is fine
when the team finds attribution useful. Plain "Claude Code" mentions in PR
bodies are also fine. The line to draw: **no `claude.ai/code/session_*` URLs
of any form**, including inside markdown links.

This applies to:

- `git commit -m` invocations.
- `gh pr create --body` and `gh pr edit --body` invocations.
- Any GitHub MCP tool that writes a PR body / comment / review (e.g.
  `mcp__github__create_pull_request`, `mcp__github__update_pull_request`,
  `mcp__github__add_issue_comment`, `mcp__github__pull_request_review_write`).
- Anywhere else you'd otherwise template the URL into shared text.

If you've previously committed or posted a body that contains the URL,
scrub it on the next edit pass — don't propagate the URL forward when you
update existing text.
